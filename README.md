# 과학기술혁신본부 업무 타임라인 대시보드

정적 HTML/CSS/JavaScript로 만든 과학기술혁신본부 업무 공개 대시보드입니다.

## 열기

공개 화면만 GitHub Pages에 올리고, 직원 수정은 별도 백엔드에 저장할 수 있습니다.

`apiBase`를 설정하면 수정 데이터는 그 백엔드로 전송됩니다.

```html
<script>
  window.OSTI_CONFIG = {
    apiBase: "https://your-backend.example.com/api"
  };
</script>
```

GitHub Pages에는 `window.OSTI_CONFIG`를 넣지 않으면 공개 읽기 전용으로 동작합니다.

로컬에서 백엔드까지 함께 돌릴 때는:

```bash
node server.js
```

그 뒤 `http://localhost:4173`을 엽니다.

직원이 저장·수정·드래그한 업무 데이터는 백엔드의 `data/dashboard-items.json`에 저장됩니다.

## 화면 구성

- 외부인: 혁신본부 주요 업무, 개시일, 완료 예정일, 업무 내용, 추진 목표를 조회합니다.
- 업무는 3개 대분류로 나누어 표시합니다.
  - 기획/정책/전략수립 업무
  - 예산/투자/심의조정 업무
  - 평가/제도/정보분석 업무
- 직원: 로그인 후 새 업무를 입력하거나 기존 업무를 수정·삭제할 수 있습니다.
- 데모 로그인 비밀번호: `osti2026`

## 데이터 기준

- 원천 자료: `260504_과학기술혁신본부 상반기 홍보계획(안).pdf`
- 구분: 주요 정책 수립, 법 제정/개정, 제도 개선, 현장 소통
- 세로축 정렬: 업무 완료 시점 오름차순
- `5월중`, `6월중`, `하반기`, `검토중`처럼 정확한 일자가 없는 항목은 화면에는 원문 라벨을 유지했습니다.
- 같은 항목의 내부 위치 계산과 정렬을 위해 `app.js`에 보수적인 임시 날짜를 넣었습니다. 해당 항목은 점선 막대로 표시됩니다.
- `초기 데이터 복원` 버튼을 누르면 PDF 기준 기본 데이터로 되돌릴 수 있습니다.
- 공개 화면만 쓰는 GitHub Pages 배포에서는 로그인과 편집이 숨겨집니다.

## 수정 위치

- 기본 일정 데이터: `app.js`의 `defaultItems`
- 직원 로그인/저장 동작: `app.js`
- 색상/레이아웃: `styles.css`
- 기본 구조: `index.html`
